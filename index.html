  3. ABSTRACT

    1. Traditional identification systems focus on gathering data from the people based on location criteria.
    2. The system is made up of fixed pattern which has weakest architecture, thus makes it vulnerable in case of storage and security of data.
    3. This system provides security to the existing system by creating unique derived from the unique patterns of the person.
    4. A system is designed in such a way that it contains 16 digit unique number which is not visible to customer as well as vendor who provides it.
    5. It leads to us to most comfortable way of using person’s identity for every government or private services.

  4. INTRODUCTION

    1. Machine Learning and Cyber Security algorithms to make next generation identification system.
    2. The system aims at producing n-digit identity for individual based on their biometrics traits because of the level of uniqueness provided by it.
    3. Machine Learning algorithm is used to extract the unique point of an individual from their biometrics.
    4. Security aspects are also considered to ensure that no attacks or threats happens. Hence, identity theft problems can be solved by using this system.

  5. EXISTING SYSTEM

    1. Many Identity proofs available but are dependent to certain constraints
    2. AADHAAR – unique number for individual
    3. Voter ID – voter’s legitimate proof
    4. Driving license –issued by state to acknowledge driving skills.
    5. PAN Card –for income tax department

  6. DRAWBACKS

    1. Centralized in nature – central point of failure causes large destruction.
    2. More Constraints are followed and consistently not accepted.
    3. High chances for identity theft.
    4. Poor infrastructure.
    5. Insecure intermediate positions.

  7. SOLUTIONS

    1. The system aims at producing n-digit identity for individual based on their biometrics traits because of the level of uniqueness provided by it.
    2. Machine Learning algorithm is used to extract the unique point of an individual from their biometrics.
    3. The data that are extracted from the biometrics are appended to form n-digit Unique ID which is represented as proof for identity.
    4. The hash value of the UID is placed on QR code to make it easy for daily usage.
    5. Security aspects are also considered to ensure that no attacks or threats happens.
    6. Hence, identity theft problems can be solved by using this system.

    2. FACE EXTRACTION

      1. It was generated based on the position of the reference point on the person’s facial image.
      2. These reference points are identified by image processing and an ID is generated based on those points in Two Dimension.

    3. FINGERPRINT EXTRACTION

      1. The fingerprint scanner is used takes 3D capture of all the ridges and valleys in the fingerprint, compared to a photo captured by an optical scanner which is a 2D image.
      2. Image processing used to identify the ridges and bifurcations in the thumb print and a static pixel value is calculated by combining four of the sibling pixels for better accounting purposes.
      3. This value is also dependent on the physical nature of the individual.

    4. UNIQUE ID GENERATION

      1. Takes data stored in cache.
      2. Computed to form N-bit UID.
      3. Permutation (if necessary).
      4. Generates two set of UID.
      5. Stores in database.

    5. HASH CODE GENERATION

      1. Blake-2B (algorithm).
      2. Truncated to 40 bit.
      3. Generates two set of hash.
      4. Stores in database.

    6. QR CODE GENERATION

      1. Quick response code – verification.
      2. Generated from hash code.
      3. Embedded in card.
      4. Scans to view details
      5. Stores in database.
